# TG Shop Bot

**Интернет-магазин в Telegram** на [aiogram 3](https://docs.aiogram.dev/) с каталогом, корзиной, заказами, гибкими ролями персонала, оплатой (включая CryptoBot) и сценарием проверки чеков.

*SQLite · один процесс · готов к запуску на VPS или ПК*

## Возможности для покупателя

| Раздел | Описание |
|--------|----------|
| **Витрина** | Категории, карточки товаров, поиск, недавние просмотры |
| **Корзина** | Количество, промокоды, бонусный счёт при оформлении |
| **Оформление** | Новая Почта (ФИО, телефон, город, отделение), доставка по городу, самовывоз |
| **Оплата** | Наложенный платёж, карта, Apple Pay / Google Pay, **CryptoBot** (счёт и авто-подтверждение при настройке); для предоплаты — фото или PDF чека |
| **Заказы** | Списки по статусам, детали заказа |
| **Профиль** | Контакты, бонусы, реферальная ссылка (если включено) |
| **Избранное** | Список товаров, уведомление при появлении на складе |
| **Отзывы** | Оценки и комментарии к купленным товарам |
| **Поддержка** | Тикеты с текстом и вложениями; ответы сотрудников в чат |

Дополнительно: напоминание о **незавершённой корзине**, ограничение по **времени работы** (поддержка и доставка по городу), режим **техработ** (витрина скрыта от клиентов, админы в панели работают).

## Возможности для администратора

- **Каталог и заказы:** товары, категории, остатки, фото, импорт/экспорт каталога (JSON), заказы, статусы, чеки (просмотр, подтверждение/отклонение).
- **Клиенты:** карточка пользователя, бонусы, **добавление в штат** (менеджер или техподдержка), снятие ролей.
- **Магазин:** промокоды, доставка, способы оплаты, витрина (приветствие, главное меню, баннер).
- **Сводка и аналитика:** метрики, списки клиентов, экспорт заказов в CSV (по правам).
- **Настройки:** уведомления, шаблоны, реферальная программа, порог низкого остатка, техработы, **права персонала** (только владелец).
- **Сервис:** скачать бэкап SQLite, **сброс БД с автобэкапом**, **загрузка БД из файла** (только владелец), при необходимости обновление с Git и перезапуск бота.

Доступ к разделам админ-меню и магазина зависит от **роли** и **назначенных прав** (см. ниже).

## Персонал и права доступа

- **Владелец и админ** — полный доступ ко всем разделам.
- **Менеджер и техподдержка** добавляются из карточки клиента кнопкой **«Добавить в штат»** с выбором роли:
  - **Менеджер** — операционная работа (по умолчанию каталог, оплата/доставка, тикеты, настройки и т.д. по пресету).
  - **Техподдержка** — по умолчанию упор на тикеты; остальное включается вручную.
- **Владелец** в **Настройки → Права персонала** выбирает роль (менеджеры / техподдержка), затем сотрудника и настраивает пункты меню (**🟢** включено / **🔴** выключено): каталог, оплата, тикеты, команда/промо, сводка, импорт/экспорт, настройки бота.
- Админ-панель в главном меню и отдельные экраны (настройки, тикеты и др.) **скрываются или блокируются**, если у сотрудника нет соответствующего права.

## Оплата через CryptoBot

- В **Настройки → Оплата** можно включить способ **CryptoBot** и задать **API-токен** [@CryptoBot](https://t.me/CryptoBot) (токен задаёт **владелец**).
- При оформлении заказа создаётся счёт в CryptoBot, клиент переходит по ссылке; оплата может подтверждаться **автоматически** фоновой задачей (`cryptobot_invoice_watcher_loop` в `satana.py`).
- Зависимость: `crypto-pay-api` (см. `requirements.txt`), логика — `utils/cryptobot_payments.py`, курсы — `utils/crypto_rates.py`.

## Сервис и база данных

В **Настройки → Сервис** доступно:

| Действие | Кто | Описание |
|----------|-----|----------|
| Скачать SQLite | Админ с доступом к настройкам | Резервная копия через `sqlite3.backup` |
| Удалить БД (бэкап и сброс) | Владелец | Сначала файл бэкапа уходит в Telegram, затем удаляется рабочая БД и создаётся пустая с таблицами |
| Загрузить БД из файла | Владелец | Отправка `.sqlite` / `.db` документом; перед заменой текущая база бэкапится и отправляется вам; затем файл подставляется и выполняется инициализация/миграции таблиц |

Рекомендуется **перезапуск процесса бота** после замены или сброса БД. Если файл на диске занят (часто Windows), бот подскажет удалить его вручную после остановки.

## Перенос каталога и фото товаров

- **Импорт каталога** из JSON (`catalog_export.json`) переносит структуру и данные; **идентификаторы фото в Telegram** (`file_id`) привязаны к **конкретному боту**.
- Если товар открывается с фото, перенесённым из другого бота, Telegram возвращает ошибку на старый `file_id`. Вместо падения бот показывает карточку **текстом** с пояснением: товар перенесён из другого бота, нужно **заново загрузить фото** в админке.

## Установка

### 1. Клонирование

```bash
git clone https://github.com/Cicadadenis/tg-shop.git
cd tg-shop
```

### 2. Виртуальное окружение

**Linux / macOS**

```bash
python3 -m venv env
source env/bin/activate
```

**Windows (PowerShell)**

```powershell
python -m venv env
.\env\Scripts\Activate.ps1
```

### 3. Зависимости

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### 4. Переменные окружения

Создайте в корне файл **`.env`** (UTF-8):

```env
BOT_TOKEN=123456789:AA...ваш_токен_от_BotFather
ADMIN_ID=123456789
```

- **BOT_TOKEN** — от [@BotFather](https://t.me/BotFather).
- **ADMIN_ID** — числовой Telegram ID владельца ([@userinfobot](https://t.me/userinfobot) и аналоги).

Если `.env` нет и бот запущен интерактивно, при старте может быть предложено создать файл (см. `data/config.py`). Альтернатива: секция `[settings]` в `settings.ini` (`token`, `admin_id`).

### 5. Путь к базе

По умолчанию: `data/botBD.sqlite`. Изменение — в `utils/db_api/sqlite.py` (`path_to_db`).

## Запуск

```bash
python satana.py
```

Остановка: `Ctrl+C`. Для сервера используйте systemd, Docker, `screen`/`tmux` и т.п.

## Полезные заметки

- **Часовой пояс:** «время работы» и напоминания о корзине зависят от **системного времени** сервера; при необходимости настройте TZ.
- **Реферальные ссылки:** `https://t.me/<username_бота>?start=ref_<код>`; username подтягивается после успешного `getMe`.
- **CryptoBot:** после смены API-токена бот может запросить перезапуск процесса для применения настроек.
- Ветка репозитория по умолчанию: **`main`**.
