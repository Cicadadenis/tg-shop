# TG Shop Bot

**Интернет-магазин в Telegram** на [aiogram 3](https://docs.aiogram.dev/) с каталогом, корзиной, заказами, ролями администраторов и сценарием оплаты с проверкой чеков.

*SQLite · один процесс · готов к запуску на VPS или ПК*

## Возможности

### Покупатель

| Раздел | Что умеет |
|--------|-----------|
| **Витрина** | Категории, карточки товаров, поиск, недавние просмотры |
| **Корзина** | Количество, промокоды, бонусный счёт при оформлении |
| **Оформление** | Новая Почта (ФИО, телефон, город, отделение), **доставка по городу** (отдельный сценарий, оплата картой), самовывоз |
| **Оплата** | Наложенный платёж, карта, Apple Pay / Google Pay — по настройке; для предоплаты можно отправить **фото или PDF чека** |
| **Заказы** | Списки по статусам, детали заказа |
| **Профиль** | Контакты для доставки, бонусы, **реферальная ссылка** (если включено) |
| **Избранное** | Список товаров, уведомление при появлении на складе |
| **Отзывы** | Оценки и комментарии к купленным товарам |
| **Поддержка** | Тикеты с текстом и вложениями; ответы админов в чат |

Дополнительно:

- напоминание о **незавершённой корзине** (фоновая задача);
- ограничения по **времени работы**: приём обращений в поддержку и оформление **доставки по городу** только в заданный интервал (время сервера);
- режим **техработ**: витрина скрыта от клиентов, админы работают как обычно.

### Администратор

- **Каталог:** товары и категории, импорт/экспорт, редактирование цены, остатка, описания, фото.
- **Заказы:** статусы, фильтры по чекам, просмотр и подтверждение/отклонение чеков, уведомления в чат и шаблоны текстов.
- **Клиенты и роли:** владелец / админ / менеджер, техподдержка, начисление бонусов, рассылка.
- **Магазин:** промокоды, способы доставки и оплаты, **настройки витрины** (приветствие, главное меню, баннер по умолчанию).
- **Аналитика и отчёты:** сводки, экспорт заказов в CSV, бэкап SQLite из настроек.
- **Сервис:** техработы одной кнопкой, напоминания о корзине, реферальные бонусы (настраиваемые суммы в БД).

## Установка по шагам

### 1. Скопируйте проект

```bash
git clone <url-репозитория> tg-shop
cd tg-shop
```

### 2. Создайте виртуальное окружение

**Linux / macOS**

```bash
python3 -m venv env
source env/bin/activate
```

**Windows (PowerShell)**

```powershell
python -m venv env
.\env\Scripts\Activate.ps1
```

### 3. Установите зависимости

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### 4. Настройте токен и администратора

Создайте в корне проекта файл **`.env`** (кодировка UTF-8):

```env
BOT_TOKEN=123456789:AA...ваш_токен_от_BotFather
ADMIN_ID=123456789
```

- **BOT_TOKEN** — выдаётся [@BotFather](https://t.me/BotFather) при создании бота.
- **ADMIN_ID** — ваш числовой Telegram ID (можно узнать у [@userinfobot](https://t.me/userinfobot) или аналога). Несколько админов через запятую в коде/настройках поддерживаются там, где это предусмотрено; основной владелец — первый `ADMIN_ID`.

> **Если `.env` нет** и бот запущен **в интерактивной консоли**, при старте может предложить ввести токен и `ADMIN_ID` и создать файл сам (см. `data/config.py`).

**Альтернатива:** в **`settings.ini`** в секции `[settings]` можно указать `token` и `admin_id` — приоритет у переменных окружения и `.env`.

### 5. Проверьте путь к базе (по желанию)

По умолчанию SQLite создаётся в `data/botBD.sqlite`. Менять расположение нужно только если вы правите `utils/db_api/sqlite.py` под свой деплой.

### 6. Запуск

См. следующий раздел.

## Запуск

Из корня проекта, с активированным venv:

```bash
python satana.py
```

В логе должно появиться сообщение о старте бота. Остановка: `Ctrl+C` в терминале.

Для **постоянной работы на сервере** используйте systemd, Docker, `screen`/`tmux` или планировщик задач Windows — главное, чтобы процесс не завершался и имел доступ в интернет к `api.telegram.org`.

## Полезные заметки

- **Часовой пояс:** интервалы «время работы» и напоминания о корзине ориентируются на **время системы**, где запущен процесс бота. Укажите нужный TZ на сервере или учитывайте смещение.
- **Реферальные ссылки:** формат `https://t.me/<username_бота>?start=ref_<код>`; username подтягивается из Telegram после первого успешного `getMe`.
- Ветка по умолчанию в репозитории: **`main`**.
